Verdict renders your UI in a real browser and checks it against WCAG contrast, type scale, spacing, and heading rules. You get a score, a fix list, and an exit code your CI can enforce.

$ npx verdict-ui check ./dist

Features

Everything a real CI gate needs

Four deterministic checks, plus the tooling to run them as a pipeline: diffing, regression gating, multi-page crawling, and a plain library API.

01

Color contrast

Real relative-luminance contrast per text node, WCAG 2.1 AA, with a suggested passing hex color.

02

Type scale

Flags font sizes that don't land on a clean scale step — usually unintentional em/rem drift.

03

Spacing grid

Detects the page's 4px or 8px base unit and flags the margins and padding that break it.

04

Heading & landmarks

One h1, no skipped heading levels, a main landmark, alt text on every image.

↗

verdict diff

50→98+48

Pixel-level screenshot diff plus a fixed / introduced / persisting issue breakdown between two renders. This is real output from bad.html → bad-fixed.html in this repo.

⟲

Regression gating

Every run appends to a local score history. verdict check fails when a change drags the score below the last passing run for that target, not only when it misses the absolute threshold.

▦

verdict crawl

Runs the full check suite against several targets in one invocation and rolls them into a single pass/fail for the build. One failing page fails the crawl.

{}

Library API

Same engine, no shell-out. import { check } from "verdict-ui" and call it from your own scripts or test suite.

◧

Browser extension

Runs the same four checks live against the tab you have open. No build output, no CLI — a direct port of the same extraction and scoring code.

↹

Agent integration (MCP)

Exposes verdict_check, verdict_diff, and verdict_crawl as MCP tools, so your coding agent calls Verdict directly instead of a human copying CLI output back in.
