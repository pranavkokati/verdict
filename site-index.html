Design QA for interfaces
built by AI agents.

Verdict renders your UI in a real browser, checks it against WCAG contrast, type scale, spacing, and heading rules, auto-fixes what's safely fixable, and catches drift across pages and viewports. You get a score, a fix list, and an exit code your CI can enforce.

Why it exists

Two ways to get an agent to build good UI

One tells the agent what good design looks like and hopes it listened. The other checks what the agent actually shipped.

Prompt injection

What most "taste" tools do

  1. Collect design guideline prompts from other sources
  2. Inject them into the agent's context before it writes code
  3. There is no step that confirms the agent followed them. You get a prompt, not a result.
Verdict

What Verdict does

  1. Render the agent's actual output in real Chromium
  2. Run deterministic checks against the computed styles and layout
  3. Return a score, a fix list, and a CI exit code. The result is something you can verify.

Four checks

Plain code, not a model call

Every check runs against the rendered DOM with ordinary, deterministic logic. Same input, same output, every time.

01

Color contrast, WCAG 2.1

Computes the real relative-luminance contrast ratio for every text node against its background. Flags anything under the AA minimum and suggests a replacement color that passes.

02

Type scale consistency

Collects every font size on the page and flags values that don't land on a clean scale step, a common sign of unintentional em or rem drift.

03

Spacing grid consistency

Checks whether margin, padding, and gap values are drawn from a consistent 4px or 8px base unit, and flags the ones that aren't.

04

Heading and landmark structure

Confirms one h1, no skipped heading levels, a main landmark, and alt text on every image.

Beyond one check

Built for a CI pipeline, not a one-off lint

Score history, visual diffing, auto-fix, multi-page crawling, multi-viewport testing, and forced interaction-state checks. The parts a real gate needs that a single prompt can't give you.

verdict diff

Prove an improvement

Pixel-level screenshot diff, plus a list of issues fixed, introduced, and still open between two renders.

verdict check

Regression gating

Fails the build when a change drags the score below the last passing run, not only when it misses an absolute threshold.

verdict fix

Auto-patch what's fixable

Finds the exact CSS declaration behind contrast, type-scale, and spacing issues via Chrome's own cascade engine and patches it directly, verified against the source before it's applied.

verdict crawl

Whole sites, one drift check

Checks several pages in one run, rolls them into a single pass or fail, and compares shared components across every page -- catching an h1 or a primary button that quietly drifted on one route.

verdict viewport

Every breakpoint

Renders one page at mobile, tablet, and desktop widths and flags horizontal overflow, undersized tap targets, and headings or landmarks a breakpoint silently hides.

verdict states

Nobody checks :hover

Forces every button and link through :hover, :focus-visible, and :active via Chrome's own DevTools protocol, and flags missing focus indicators and contrast that only breaks on interaction.

Get started

Two commands and no configuration

Works against a local build, a running dev server, or a live URL. No account and no server to run.

Install # install
npm install verdict-ui
npx playwright install chromium

# check a build
npx verdict-ui check ./dist/index.html
CI # .github/workflows/verdict.yml
- run: playwright install --with-deps
- run: npx verdict-ui check ./dist
    --threshold 80

# exits 1 on fail, blocks the PR
